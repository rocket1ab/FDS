# H1-H7 飞机光辐射点燃与毁伤评估项目交接

更新时间：2026-07-23

## 1. 我们正在做什么

本项目研究核爆平行光热辐射入射飞机后，从体素法 DDA 遮挡追踪、外部热流赋予、FDS 点燃与燃烧模拟，到材料、设备和整机毁伤树评估的完整流程。

当前模型是新版 H1-H7 飞机模型。主要输入为：

- 光冲量 Q，统一使用 J/cm²。
- 核爆当量 W，单位 kt；当量决定热脉冲时间尺度。
- 方位角 az 和俯仰角 el。
- 当前主要基线角度为 az=270°、el=15°，当量为 W=100 kt。
- FDS 正式案例目标时长为 T_END=1500 s。

当前核心目标有三个：

1. 确定使新版模型 17 个材料/设备组全部达到重度毁伤的最低光冲量范围。
2. 在光冲量不变时研究角度、当量、脉冲宽度及有物理依据的燃烧参数对毁伤的影响。
3. 开展网格分辨率一致性与固定 Q 的当量独立性验证，并形成可复核的 Markdown 报告、曲线和毁伤树。

## 4. 光冲量归一化必须遵守的规则

旧方案曾错误地把标称 Q=100 J/cm² 生成为最大局部积分光冲量约 28.86 J/cm²。该旧热流缩放方案已经退役，禁止恢复或混入阈值结论。

当前正确关系为：

`Q = 0.1 * E0 * integral(F(t) dt)`

其中 E0 单位为 kW/m²，Q 单位为 J/cm²。当前 Q=100 的修正案例最大局部积分光冲量约为 97.14 J/cm²，低于 100 的小幅差异来自角度投影、遮挡和空间离散，而不是错误缩放。

固定 Q 改变当量时必须同时执行：

1. 按 `t_max = 0.044 * W^0.42` 缩放 NUCLEAR_RAMP 时间轴。
2. 按时间缩放系数的倒数缩放所有 EXTERNAL_FLUX 峰值。
3. 重新积分验证入射面 Q 保持不变。

禁止只改 RAMP 时间而不改峰值，也禁止只改峰值而把总光冲量一并改变。

### 7.3 网格一致性判定

不能只比较“整机是否重度”。至少比较：

- 17 组逐项毁伤等级。
- 各组峰值温度。
- 超过轻度、中度、重度阈值的连续持续时间。
- 全域 HRR 峰值与时间积分。
- 点火顺序和续燃行为。
- 毁伤树系统节点和整机等级。

建议关键连续量相对差不超过 10%，且逐组等级和关键系统结论不改变，才可称为工程上一致。当前只有粗网格与现网格两档，这只能做一致性筛查。正式“网格无关性”结论还应对代表工况增加细网格，形成粗、中、细三档并做收敛分析。

### 7.4 当量独立性判定

对 W=10、50、200、500、1000 kt 比较：

- t_max、RAMP 积分和 EXTERNAL_FLUX 峰值。
- 入射面和最大局部积分光冲量是否仍为 Q=50 对应值。
- 各设备峰值与持续时间。
- 可燃材料是否点燃、是否续燃及 HRR 差异。
- 严格重度毁伤数和毁伤树等级。

若结果随当量明显变化，应表述为“固定总光冲量下的脉冲形状/峰值效应”，不能说光冲量不同。

## 8. 绝对不要再踩的坑

### 8.1 不得混用错误热流缩放

旧 v4 或旧 legacy-flux 案例的标称 Q 与实际积分 Q 不一致。它们可用于追溯问题，但不能与修正 Q-normalized 案例混成阈值序列。

### 8.2 不得用 RADM 代替雷达电子系统

RADM 只代表雷达罩。雷达罩毁伤可通过透波性能和结构变形影响雷达功能，但不能直接当作雷达电子设备温度证据。

### 8.3 不得把 BED 的 THICKNESS 当床垫整体厚度

当前 BED 的 THICKNESS 是尼龙覆盖层厚度，审查值为 0.89 mm。旧案例把它设得过厚会显著增加热惯性，导致难点燃、难升温。

### 8.4 不得为制造全毁伤随意削弱材料

不得降低毁伤阈值、随意降低点燃温度、添加不存在的热源或无依据地改变材料参数。自适应方案必须单因素、有来源、单独命名，并与同 Q 基线比较。

### 8.5 不得把瞬时跨探针最大值说成固定探针曲线

材料温度包络是每个时刻对该组所有有效探针取最大值。它不是某一个固定探针的时间历程。报告中必须同时说明包络定义和固定单探针峰值。

### 8.6 不得把快照说成完成

- 运行中有 DEVC：实时快照。
- 异常停止但有效模拟时间达到 1000 s：长时快照。
- 达到 1500 s 且日志含正常 STOP：正常完成。

### 8.7 不得只看峰值而忽略持续时间

毁伤判据是温度和持续时间的组合。峰值达到阈值但持续时间不足，不能判为相应等级。

### 8.8 不得重复提交队列

先检查节点 lock、状态 JSON、等待器 PID 和实际 FDS 进程。重复启动会造成同一案例并发、结果文件互相覆盖和节点过载。

### 8.9 不得覆盖旧案例目录

所有新方案必须建立新目录和新 CHID。输入、输出和评估文件应保留，便于追溯。

### 8.10 不得使用过长案例名

Windows 路径长度曾导致复制 `.fds` 失败。新验证案例已经使用短名称，完整来源和变更写入 `case_summary.json`。不要恢复超长后缀。

### 8.11 不得假定 node02 可用

当前网关到 node02 返回 SSH 认证拒绝。除非管理员修复，不要反复提交。也不要因 node02 不可达而干扰其它节点的无关任务。

### 8.12 不得用两档网格直接宣称严格网格无关

粗网格与现网格一致只能说明当前结论对一次粗化不敏感。正式网格无关性通常需要至少三档网格和收敛性论证。
